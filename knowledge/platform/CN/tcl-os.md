# TCL OS 平台适配规范 (CN)

本规范定义了 TCL 智能电视系统（针对中国区市场，基于灵悉 UI V5.2 架构）的界面交互、导航设计、界面结构与限制条件。

## 1. 导航与焦点规则
- **交互载体**：完全依赖 D-pad 遥控器（上、下、左、右、确认、返回/Esc）。
- **焦点设计**：
  - **聚焦态**：聚焦元素必须具有极强的视觉辨识度。推荐规范：“发光外边框（青蓝色 `#00e5ff`） + 呼吸光晕 + 缩放 (1.06x) + 适当投影”。
  - **默认焦点**：首个进入页面时，默认焦点必须停留在页面最核心的操作元素（如第 1 行第 1 个卡片，或设置列表的第 1 项）。
  - **丢失防范**：在切换数据状态或打开/关闭抽屉时，如果被删除元素当前拥有焦点，必须由 JS 主动转移焦点到相邻的可聚焦元素，严禁发生焦点隐藏或丢失。

## 2. 页面布局与安全区
- **物理分辨率**：基准设计分辨率为 1920x1080 (FHD)。
- **安全区域 (Safe Area)**：
  - 边缘保护：四周必须留出至少 **6% (左右各 115px，上下各 65px)** 的安全边距，避免物理切边阻挡。
  - **文字溢出**：对限制宽度的卡片或菜单，当文本超长时应在行尾采用 `...`（省略号）截断，并在选中（Focused）时支持跑马灯式水平滚动显示。

## 3. 系统设置结构
- **三栏级联设置**：
  - **一级菜单（主类别）**：常驻左侧（如“图像”、“声音”、“系统”）。
  - **二级菜单（设置项）**：位于中侧，包含各个功能的具体控制器（如“图效模式”、“对比度滑块”、“护眼开关”）。
  - **三级菜单 / 浮层（高级微调）**：右侧展示当前选中项的详细描述、属性修改或点击后弹出的高级对话框。
- **三合一快捷面板**：
  - 悬浮于屏幕下侧或侧边，用于在播放视频时快速调节画质或声音。包含快捷网格（Switch）和调节条（Slider）。

## 4. 弹窗与 Dialog 规范
- **焦点锁定 (Focus Trap)**：Dialog 弹出后，所有的 D-pad 操作必须被拦截并锁定在 Dialog 内部（如“确定”、“取消”按钮之间），按方向键绝对无法移出 Dialog 到背景层。
- **返回行为**：按返回键默认关闭 Dialog，并把焦点物交还给弹出 Dialog 之前的触发节点。
- **视觉位置**：Dialog 必须居中覆盖在屏幕上方，背景包含高斯模糊。

## 5. 状态与异常设计
- **Grey State（不可用状态）**：当选项因条件限制（如无信号源输入、特定画质算法不支持当前视频源）不可用时，按钮显示为灰色（字色 `#646a75`），此时该元素应**跳过焦点**或**允许聚焦但展示无法开启的 Toast 提示**（灵悉 UI 推荐后者，以保证用户知道该选项的存在）。
- **Toast 提示**：提示窗在底部居中短暂显示（停留 2.5s），不夺取当前操作的焦点。
- **Loading 与重试**：网络相关操作提供圆形加载指示器；超时则显示“网络已断开，请重试”的错误浮层，并提供聚焦的“重试”按钮。
