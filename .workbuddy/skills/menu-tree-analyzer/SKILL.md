## 职责介绍

仅在确实需要时载入系统菜单树，分析用户功能在设置层级中的合理归属，推荐最符合用户直觉的插入点，并以清晰的 Diff 形式展示层级结构的变化。

## 本地工具与 MCP 调用指南

为了高效、准确地检索数万条菜单数据，AI 助手**必须优先检查并使用内置的 MCP 服务工具，如果 MCP 未激活，则退回到本地命令行工具**。

### 1. 优先使用 MCP 工具（当已配置并激活时）
如果配置了 `product-prototype-kit` MCP 服务器，AI 助手**必须直接使用以下 MCP 原生工具进行检索**：
- **`search_menu`**：用于模糊搜索关联节点。
  - *参数*：`region`, `query`, `platform` (可选)。
- **`get_menu_path`**：用于获取节点完整路径与详细属性。
  - *参数*：`region`, `nodeId`。
- **`diff_menu`**：用于生成菜单层级变更的差分 Diff 文本。
  - *参数*：`region`, `parentNodeId`, `action`, `targetName`。

### ⚠️ 检索准确性与零幻觉约束（最高优先级）
- **禁止凭空捏造**：AI 助手在进行菜单树检索和设计提案时，**必须完全基于 MCP 或 `menu-helper.js` 从数据库返回的真实节点 ID 和路径，绝对不允许有任何胡乱编造、拍脑袋编造路径的行为**。
- **杜绝 AI 幻觉**：如果数据库中查不到某项设置对应的父节点，必须如实向用户反馈，或询问用户是否要新建相关父节点，**绝不允许产生幻觉或凭空构造非官方菜单树中的不存在的路径**。
- **中译英检索约束（海外项目）**：如果当前选择或命中的是 **海外项目 (Overseas)**，但用户在输入需求或描述菜单时使用了 **中文**（例如：“在海外图像设置下添加‘高级设置’”），AI 助手**必须首先将中文菜单名称翻译为准确的英文对应词（如“图像”->“Picture”，“高级设置”->“Advanced Settings”），然后使用英文去检索海外菜单树数据库，绝对禁止直接用中文去检索海外菜单树，也绝对禁止去检索国内菜单树**。

### 2. 本地命令行工具（当 MCP 未激活时的备用方案）
如果 MCP 服务未开启，必须运行本地 `scripts/menu-helper.js` 进行检索：
- **模糊搜索关联节点**：`node scripts/menu-helper.js search <cn|overseas> "<关键字>"`
- **获取节点完整路径**：`node scripts/menu-helper.js get-path <cn|overseas> "<节点ID>"`
- **辅助生成差分 (Diff)**：`node scripts/menu-helper.js diff <cn|overseas> "<父节点ID>" <add|delete|modify> "<目标项名称>"`

## 触发规则（防冗余过滤）

**重要规则**：并非所有需求都需要读取和处理菜单树。

### 1. 允许加载菜单树的场景：
- 新增菜单入口
- 修改已有菜单名称或属性
- 调整菜单层级结构与父子关系
- 修改或重定向设置项的跳转路径

### 2. 禁止加载菜单树的场景：
- 独立的独立应用（APK）设计（例如“拍照校准 APK 设计”）
- 纯画质/音质算法参数优化
- 内部底层参数逻辑调整（非前台菜单项调整）
- 桌面（Launcher）卡片及横幅内容修改

*在禁止加载的场景下，AI 应跳过菜单树读取，以减少上下文干扰并提升运行速度。*

## 分析与输出规范


当判定需要进行菜单分析时，本 Skill 必须输出以下内容：

### 1. 路径推荐对比
清晰对比当前菜单路径与推荐新增后的路径：
- **当前路径示例**：`Settings > Picture > Advanced Settings`
- **推荐新增路径示例**：`Settings > Picture > Advanced Settings > [New Feature Name]`

### 2. 推荐合理性阐述
必须从产品体验角度说明推荐理由，如：
1. **符合用户认知**：该功能属于画质相关，与 Picture 目录下其他项强关联。
2. **保持层级扁平**：不新增一级菜单，维护设置页整体简洁度。
3. **低学习成本**：利用已有父节点，用户可根据旧有路径直观找到。

### 3. 菜单差分 (Menu Diff)
使用标准 Diff 格式标记菜单变化：
- `+` 表示新增菜单项
- `-` 表示删除/下线菜单项
- `→` 表示菜单路径移动或名称修改

**Diff 示例**：
```diff
  Settings
    Picture
      Advanced Settings
+       Color Calibration (新功能)
-       Old Legacy Mode (废弃模式)
        Brightness → Panel Brightness (更名)
```
